# Custom Rule for Implementation Plans

Whenever I am asked to create an `implementation_plan.md` artifact (while in PLANNING mode), I must strictly include a new top-level section titled "## Dependencies Used". 

This section must list:
1. Every specific local skill (e.g., `flutter-clean-architecture`) that heavily influenced the plan.
2. Every MCP Server (e.g., `dart-mcp-server`) that will be leveraged during the execution phase. 

This ensures that the user is completely aware of the context and tools I am bringing into the task.
