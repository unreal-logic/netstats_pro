# Flutter Document Generation

## When to Use This Skill
- When providing a post-game Match Report download feature.
- When generating Invoices for SaaS subscription billing.
- When exporting a formatted view of player or team analytics.
- When the user needs to physically print a page from the app.

## How to Use This Skill
1. **Setup:** Add the `pdf` and `printing` dependencies to the project using `mcp_dart-mcp-server_pub`.
2. **Document Structure:** Import `package:pdf/widgets.dart` as `pw` to avoid collisions with Flutter's Material widgets. 
3. **Build Pages:** Build the PDF layout using `pw.Document()`, `pw.Page()`, and standard layouts like `pw.Column`, `pw.Row`, and `pw.Table`.
4. **Handle Fonts:** If using custom typography, load the font data (e.g., from assets or network) and explicitly pass it into the PDF rendering context, as standard Flutter fonts will not automatically render in the PDF engine.
5. **Output:** Use `Printing.layoutPdf` to open a system print dialog or `Printing.sharePdf` to export the generated byte array.

## Conventions
- Always isolate PDF generation logic into its own service class away from the UI rendering layer.
- Keep PDF file generation off the main thread where possible (using `compute`) to avoid jank when generating very large reports.
- **MCP Integration:** Use the `dart-mcp-server` to find and add PDF dependencies, and to read PDF raw files if debugging byte outputs is necessary.
